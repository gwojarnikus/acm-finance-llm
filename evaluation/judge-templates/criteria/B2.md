# Criterion B2: Causal Chain Robustness

## Definition

Are conclusions causally grounded in stated premises? Each conclusion in the recommendation should trace back to specific premises in the analysis. Detect *non sequiturs* (conclusions that don't follow from premises) and unjustified logical leaps.

This addresses "hallucinated logic": grammatically correct conclusions that don't actually follow from the data. Wei et al. (2022) on chain-of-thought reasoning highlights the importance of explicit reasoning chains.

## Behavioral anchors (from BARS pilot v1)

- **Level 0:** Non sequitur — conclusions completely disconnected from analysis ("therefore X" with no supporting argument).
- **Level 2 (poor):** Multiple unjustified leaps; conclusions appear without traceable premises.
- **Level 5 (average):** Most conclusions justified, but 2–3 logical gaps (steps without intermediate link).
- **Level 8 (good):** Solid logic; each conclusion references specific premises. Minor stylistic gaps tolerable. Observed in R1 (e.g., S1-R1-C line 81: `A universal turnover tax... is much harder to reconcile with the current EU framework, not because the EU mandates Poland's present PIT/CIT design, but because the combination of VAT-harmonisation logic, state-aid/internal-market scrutiny, and especially the Union's Pillar Two minimum-tax directive makes a clean one-tax substitution legally incomplete` — conclusion grounded in 3 explicit premises).
- **Level 10 (excellent):** Plus explicit "because... therefore... which implies... finally..." chains with named logical steps; meta-acknowledgment when a step is weaker than others.

## Scoring method

**Type:** causal tree construction.

**Procedure:**

1. **Conclusion inventory.** Identify the substantive conclusions in the report's Recommendation / Key Conclusions section. List them.
2. **Premise tracing.** For each conclusion, trace back to the Analysis / Data Verification sections. Can you identify specific premises that support the conclusion?
3. **Compute ratio:** `grounded_conclusions / total_conclusions`. Initial score = `10 × ratio`.
4. **Adjustments:**
   - Subtract 1 for each *non sequitur* (conclusion with no traceable premise).
   - Subtract 0.5 for each logical leap (premise to conclusion missing intermediate step).
   - Add 1 (cap at 10) if the report explicitly enumerates its reasoning chain step-by-step.

## What counts as a "conclusion"

- Recommendations ("Do X", "Do not pursue Y").
- Empirical claims that summarize analysis ("This implies fiscal infeasibility").
- Comparative judgments ("A is preferable to B").

What does NOT count as a conclusion:
- Restatement of facts ("PIT revenue was X billion").
- Definitional statements ("UBI means...").

## What counts as "grounded in premises"

The conclusion can be traced back to specific data points, citations, or reasoning steps in the Analysis section. Direct quote-back is not required, but a reader should be able to find the supporting evidence.

## Asymmetry per response type

- Apply uniformly. All response types must demonstrate causal reasoning.
- **AKM** has step-JSON dialogue traces showing how reasoning evolved across iterations — but this is evaluated in the separate process template, NOT here. For B2 in P2 mode, evaluate ONLY the final report.
