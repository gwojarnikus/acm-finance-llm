# Criterion A2: Source Auditability

## Definition

Are the cited sources real (not fabricated) and do they actually contain the data attributed to them?

This criterion addresses the well-documented LLM problem of source fabrication: plausible-looking citations to non-existent papers, fictitious authors, or URLs that lead to unrelated content (Faisal & Tunkel, 2025; Kotonya & Toni, 2020).

## Two-level evaluation

A2 is scored in two layers because full content verification requires tool use that may be unavailable.

### Level 1: Formal verification (max 6 points, no tool use required)

For each source in the report (sample of up to N=10 if the report has many), check the following formal properties:

| Check | Pass = +1 | Fail = 0 |
|-------|-----------|----------|
| Author or organization name present | yes | no |
| Title present | yes | no |
| Year present | yes | no |
| URL or DOI present | yes | no |
| URL has plausible structure (e.g., `https://domain.tld/...`, not `https://example.com/...`) | yes | no |
| Access date or publication date specified | yes | no |

Compute Level 1 score as: `level_1 = 6 × (passing_sources / total_sampled_sources)`.

### Level 2: Content verification (additional +0 to +4 points, tool use OPTIONAL)

If tool use (WebFetch) is available, randomly sample 3 sources from the report and:

1. Fetch the URL.
2. Verify that the fetched content plausibly contains the cited fact (date, statistic, claim).
3. Score: `level_2 = 4 × (verified_sources / 3)`.

If tool use is unavailable, set `level_2 = 0` and add `"level_2_skipped": "tool_use_unavailable"` to your output.

### Final score

`score = level_1 + level_2`, capped at 10, rounded to 0.5 increments.

## Behavioral anchors (from BARS pilot v1)

- **Level 0:** All sources are fabricated or completely unverifiable (e.g., authors that don't exist, journals that don't exist). Hypothetical; not observed in R1. **Soft Veto trigger.**
- **Level 2 (poor):** ≥50% of sources fail formal checks OR fabrication detected (URL leads to unrelated content, author fictitious). Hypothetical for R1; typical of poor B1 outputs.
- **Level 5 (average):** All sources pass formal checks, but no content verification performed; some URLs broken or generic.
- **Level 8 (good):** All sources formally valid, 2/3 sampled URLs verify their cited facts. Observed in R1 (e.g., S2-R1-C citations to FRED, Federal Reserve, IMF — all canonically formatted to recognized institutions).
- **Level 10 (excellent):** Full formal validity + all 3 sampled URLs verify + access dates cross-checked with publication dates.

## Fabrication detection (special flag)

If you detect a fabricated source — e.g., an author name that is implausible, a journal that does not exist, or a URL that obviously could not lead to the cited content — set `fabrication_detected: true` in your output. This triggers Soft Veto regardless of formal score.

Examples of fabrication signals:
- Author "John P. Smith" cited for a Polish tax law analysis (no plausible expertise).
- Journal title like "International Journal of Generic Economics" with no verifiable existence.
- URL like `https://gov.pl/random-document.pdf` with no real path.
- Citation `LinkedIn/PGTAX (2026)` for a fiscal analysis — informal source masquerading as authoritative.

## Asymmetry per response type

- **AKM:** Sources section in APA 7 with full bibliographic details — easier to formal-verify.
- **B1:** Citations may be inline only; less formal structure to verify against.
- **R1:** `citeturn` references in R1-C are pseudo-citations to web search results — count as sources but note their non-standard format. R1-G uses numbered footnotes with References section.

## Limitation acknowledgment

If you cannot perform Level 2 verification (no WebFetch tool), state this explicitly in the output. Max score is then 6 (Level 1 only). This is not a failure of the report; it's a limitation of the evaluation environment.
