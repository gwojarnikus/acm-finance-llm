# Criterion D5: Metric Selection Suitability

## Definition

Are the analytical tools / metrics chosen appropriately for the asset class / problem at hand? Using P/E for a REIT instead of FFO is a fundamental domain error; using DCF for a near-bankrupt company without liquidation value is similarly wrong.

This criterion tests domain knowledge: even with correct arithmetic (D4), using an inappropriate metric produces meaningless results.

## Pre-requisite: per-case briefing

This criterion requires a per-case D5 briefing listing the suitable metrics for the case. If not provided, set `score: null` and `skipped_reason: "per-case D5 briefing not provided"`.

**Per-case expected metrics** (loaded into `{briefing_D5}`):

For S1 (PL turnover tax):
- **Tax incidence:** statutory rate, effective rate, marginal rate
- **Behavioral response:** Laffer curve, price elasticity of demand/supply, behavioral response coefficients
- **Distributional:** Gini coefficient, deadweight loss, equivalent variation
- **Sectoral impact:** sector-specific elasticities, pass-through coefficients
- **Comparative:** cross-country effective tax rates, EU averages
- **NOT suitable:** simple flat-rate calculations without behavioral response (oversimplification)

For S2 (pension fund allocation):
- **Liability matching:** duration, key rate duration, convexity
- **Real returns:** real return (vs nominal), surplus risk
- **LDI:** glide path, hedge ratio
- **Inflation:** breakeven inflation, real yield, CPI-linked structures
- **Risk-adjusted:** Sharpe ratio (in real terms), Sortino, max drawdown
- **NOT suitable:** simple nominal yield comparisons ignoring inflation

For S3 (Italy UBI):
- **Welfare economics:** poverty gap, poverty headcount, equivalence scale
- **Labor:** labor supply elasticity, reservation wage, participation rate
- **Fiscal:** fiscal multiplier, automatic stabilizer effect, dependency ratio
- **Demographic:** old-age dependency ratio, working-age population
- **NOT suitable:** simple per-capita transfers without behavioral response

## Behavioral anchors (from BARS pilot v1)

- **Level 2 (poor):** Clearly inappropriate metric (e.g., P/E for REIT, GDP for individual welfare).
- **Level 5 (average):** Appropriate metric category, but unspecified or generic choice.
- **Level 8 (good):** Suitable metrics chosen with explicit justification. Observed in R1 (e.g., S2-R1-C line 39: `TIPS index has strong positive exposure to core inflation shocks. This is crucial because much of the current inflation pulse... is still visibly energy-heavy, but a five-year 4–5% regime would almost certainly require broader core persistence` — explicit metric selection (TIPS) + rationale (core vs energy inflation)).
- **Level 10 (excellent):** Plus comparison with alternative metrics and argument for the chosen one. Observed in R1 (e.g., S2-R1-C line 45: compares commodities (tactical insurance) with TIPS (strategic hedge) with explicit argument why they don't substitute).

## Scoring method

**Type:** per-case briefing cross-check.

**Procedure:**

1. **Metric inventory.** From the per-case briefing, identify the expected suitable metrics.
2. **Coverage check.** How many of the expected metrics does the report use substantively? (Mention is not enough — use in analysis.)
3. **Suitability check.** Are the metrics chosen appropriate for the specific question? (Right category + right specific metric.)
4. **Justification check.** Does the report explain why this metric was chosen?
5. **Alternative comparison.** Does the report consider alternative metrics and explain why they were not chosen?

Score guidance:
- ≥3 suitable metrics used substantively + justification + alternative comparison → 9–10
- ≥3 suitable metrics + justification → 7–8
- 2 suitable metrics + partial justification → 5–6
- 1 suitable metric or generic choice → 3–4
- Inappropriate metric chosen → 0–2
