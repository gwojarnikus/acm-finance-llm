# Criterion C5: Process Auditability

## Definition

Does the report allow a reader to reconstruct the reasoning trace from data to conclusion? Transparency of methodological decisions: what assumptions were made, what data was selected and why, how were values aggregated, where do specific numbers come from.

This criterion is grounded in the Explainable AI (XAI) literature (Arrieta et al., 2020; Faisal & Tunkel, 2025; Yeo et al., 2025), adapted to the textual product. In financial decision-making, audit trails are essential for compliance and trust — a report whose conclusions cannot be retraced is not actionable in regulated environments.

## Behavioral anchors (from BARS pilot v1)

- **Level 2 (poor):** No methodology disclosed; results presented without explanation of the path that produced them ("black box").
- **Level 5 (average):** Partial transparency — method described but key parameters omitted ("we used standard valuation methods").
- **Level 8 (good):** Full disclosure of assumptions, parameters, and methodological choices. Observed in R1 (e.g., S1-R1-C line 43: `To broaden the base, one can combine the 2025 sales figure for the 10+ segment with Statistics Poland's latest micro-enterprise revenue data and with the very stable sales-to-total-revenue ratio observed in the 10+ enterprise data, which was 96.6% in 2024 and 96.5% in 2025` — full methodological trail with parameters).
- **Level 10 (excellent):** Plus explicit sensitivity analysis showing where alternative methodological choices would change conclusions. Observed in R1 (e.g., S1-R1-C line 45: `if the tax were imposed only on the 2025 net sales of the currently observed 10+ worker enterprise population, the neutral rate would jump to roughly 4.9%` — explicit sensitivity).

## Scoring method

**Type:** methodology check + marker search.

**Procedure:**

1. **Methodology section check.** Is there an explicit Methodology, Methods, Approach, Assumptions, or "How we did this" section? If not, are methodological decisions integrated into the analysis sections?
2. **Marker scan.** Look for transparency markers:
   - "we assume", "this estimate uses", "our parameter X = Y because"
   - "the calculation is based on", "the method involves"
   - "the data source is", "we selected X over Y because"
   - "for simplicity we assume", "sensitivity to this assumption is"
3. **Numerical traceability.** For each numerical result, can a reader trace back to: (a) the data inputs, (b) the formula or method, (c) any assumptions made?
4. **Sensitivity analysis.** Does the report identify where alternative methodological choices would change conclusions?

Score guidance:
- All assumptions disclosed + sensitivity analysis present → 9–10
- All assumptions disclosed + parameters explicit → 7–8
- Methodology described, some parameters implicit → 5–6
- Some methodology mentioned → 3–4
- Black box / no methodology → 0–2

## R1 self-reported metadata handling

R1 reports include a `Research Process Metadata` block with `search_queries_used`, `confidence_justification_reported`, and `sources_consulted_estimated`. **These do NOT count for C5.** Score based on the body of the report — does the body itself expose methodology, or does it hide everything behind self-reported claims?

For AKM, the underlying step-JSON files with `_dbg_*` fields provide system-level auditability that is NOT visible to the C5 scorer (those are evaluated in process-mode template). C5 in P2 mode scores only what the final report exposes.

## Asymmetry per response type

- **AKM:** structural process auditability via step-JSON-y (out of scope here). For the final report, depends on whether the Moderator includes methodology explicitly.
- **B1:** depends on base model's tendency to explain reasoning.
- **R1:** observed in pilot — high C5 baseline because Deep Research naturally exposes search queries and source aggregation methodology.
