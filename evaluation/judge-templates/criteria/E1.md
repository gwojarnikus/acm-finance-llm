# Criterion E1: Format Compliance

## Definition

Does the report comply with the format schema mandated by the source prompt for its type (AKM / B1 / R1)? Different system types have different format expectations defined in their respective prompts.

## Per-type schema (loaded into `{format_schema_E1}`)

**AKM** (from `prompts/agents/akm-prompts-en.md`):
- Header: `**[ANALYST]**` (mandatory)
- Section 1: `## 1. Problem Diagnosis`
- Section 2: `## 2. Data Verification (Grounding)` with numbered list of facts + `**Sources:**` block in APA 7
- Section 3: `## 3. In-Depth Analysis`
- Section 4: `## 4. Recommended Strategy / Key Conclusions` (or similar — must address recommendation)
- Markdown formatting throughout
- APA 7 citations in `**Sources:**` block

**B1** (from baseline prompt — looser):
- No mandatory header
- Sections may be free-form
- Citations may be inline in body text
- Markdown formatting recommended but not mandated

**R1** (from `prompts/reference/r1-prompt-template.md`):
- Deep Research natural structure (Problem Diagnosis, Data Verification, Analysis, Recommendation typically)
- `## Research Process Metadata` block at end with self-reported fields (`sources_consulted_estimated`, `search_queries_used`, `confidence_level_pct`, `confidence_justification`, `key_data_limitations`, `output_word_count_estimated`)

## Behavioral anchors (from BARS pilot v1)

- **Level 2 (poor):** Breaks fundamental schema (wrong sections, no headers, plain text).
- **Level 5 (average):** Schema partially observed (some sections present, others missing or reordered).
- **Level 8 (good):** Schema observed; minor deviations. Observed in R1 (all R1 reports follow Deep Research structure + `Research Process Metadata` block).
- **Level 10 (excellent):** Full compliance + additional helpful elements (Bottom Line section, tables for comparison, executive summary). Observed in R1 (e.g., S1-R1-C lines 83–91: explicit `Bottom Line` with bullet points — transparent executive summary).

## Scoring method

**Type:** schema validation per type.

**Procedure:**

1. **Load expected schema** for the report's type (from `{format_schema_E1}` field above or fetched from rubric JSON `evaluation_method.schemas_per_type`).
2. **Check mandatory elements.** Each element passing = 1 point (max 8 for full compliance).
3. **Check bonus elements** (Bottom Line, executive summary, comparison tables, well-organized markdown hierarchy). Each = +0.5 (max +2).
4. **Penalty for major deviations** (-1 each):
   - Missing required section
   - Citations in wrong format (e.g., not APA 7 when AKM)
   - Free text where structured content expected

Score guidance:
- Full compliance + bonus elements → 9–10
- Full compliance, no bonuses → 7–8
- Most elements present, minor issues → 5–6
- Several elements missing → 3–4
- Schema broken → 0–2

## Per-type asymmetry

- **AKM:** strictest schema, easiest to verify. Missing `**[ANALYST]**` header or `**Sources:**` block is a clear -1 each.
- **B1:** loose schema, harder to penalize. Focus on basic markdown usage.
- **R1:** moderate schema; check for `Research Process Metadata` block.
