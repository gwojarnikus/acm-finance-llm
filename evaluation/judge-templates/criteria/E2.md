# Criterion E2: Negative Constraints

## Definition

Does the report respect the negative constraints (prohibited actions) specified in the source prompt for its type? Testing model's ability to NOT do specific things is often harder than testing positive instructions.

## Per-type checklist (loaded into `{checklist_E2}`)

Different response types have different prompts with different prohibitions.

### AKM (from `prompts/agents/akm-prompts-en.md`)

- ☐ No back-references ("as before", "as stated earlier", "as we discussed")
- ☐ No signaling response to critic (e.g., "in response to the Critic's feedback")
- ☐ Language of response = language of question (English question → English response)
- ☐ All citations in APA 7 format
- ☐ "Today is {{date}}" anchoring present and consistent with content
- ☐ Sources section complete and at the end
- ☐ No language mirroring of search results (response language ≠ search result language)

### B1 (minimal — single-shot prompt, fewer constraints)

- ☐ Language of response = language of question
- ☐ Citations have author + year (any format)

### R1 (from `prompts/reference/r1-prompt-template.md`)

- ☐ Self-reported metadata block at end
- ☐ `confidence_reported_pct` in [0, 100]
- ☐ `key_data_limitations_reported` ≥ 1 statement
- ☐ Research Process Metadata block present
- ☐ Sources count reported

## Behavioral anchors (from BARS pilot v1)

- **Level 2 (poor):** Multiple violations of checklist items.
- **Level 5 (average):** 1–2 violations (e.g., a back-reference, or APA 7 sloppy).
- **Level 8 (good):** No checklist violations. Observed in R1 (all R1 reports comply with `r1-prompt-template.md`: self-reported metadata, confidence_reported_pct in range, key_data_limitations ≥ 1).
- **Level 10 (excellent):** Plus active signaling of constraint compliance (e.g., explicit "Note: data as of [date]"). Observed in R1 (e.g., S2-R1-G line 109: `Current market data are early-May 2026 snapshots and can move quickly` — explicit signaling).

## Scoring method

**Type:** checklist per type.

**Procedure:**

1. **Identify the response type's checklist** from `{checklist_E2}` above.
2. **For each checklist item, check the report:**
   - Pass: 1 point per item (max = checklist size)
   - Fail (violation): -1 from baseline 10
3. **Bonus for active signaling:** +1 if report explicitly notes its constraint compliance (e.g., "Note: data as of [date]", "Following the instruction to avoid back-references...")
4. Normalize to [0, 10].

Score guidance:
- All checklist items passed + active signaling → 9–10
- All checklist items passed → 7–8
- 1 minor violation → 5–6
- 2–3 violations → 3–4
- Multiple violations → 0–2

## Critical instruction

When evaluating a report, you must apply the CORRECT checklist for its response_type. If the report is AKM, use the AKM checklist (7 items). If B1, use B1 (2 items). If R1, use R1 (5 items).

Do NOT apply AKM rules to B1 or vice versa — each system has different prompts and different constraints.

## Asymmetry per response type

- This criterion IS the asymmetry — each type has its own checklist with different number of items. The score is normalized to [0, 10] regardless of checklist size.
