# Criterion A3: Epistemic Uncertainty Calibration

## Definition

Does the report distinguish hard facts from estimates? Does it use probabilistic language ("approximately", "likely", "around", "between X and Y") where data is incomplete or projected? Does it explicitly acknowledge limitations?

In financial analysis, overconfidence is dangerous: presenting an estimate as certainty can lead to poor decisions (Tonmoy et al., 2024). Conversely, hedging everything with "may" and "could" obscures actionable information. Good calibration is precision-matched modality: hard data presented confidently, estimates clearly marked.

## Behavioral anchors (from BARS pilot v1)

- **Level 2 (poor):** All figures and projections presented as certain. No words like "approximately", "likely", "estimated". No limitations section. Hypothetical for R1.
- **Level 5 (average):** Some estimates flagged with modality ("about 3.6%"), but multi-year projections presented deterministically. Inconsistent application.
- **Level 8 (good):** Each estimate carries explicit uncertainty marker. Sensitivity ranges given where appropriate. Explicit Limitations section addresses key unknowns. Observed in R1 (e.g., S1-R1-C line 43: `a flat turnover-tax rate of roughly 3.5% to 3.65%. A practical point estimate is therefore about 3.6%. This estimate assumes no exemptions, no behavioural change, and ordinary compliance` + lines 110–114 listing 4 explicit data limitations).
- **Level 10 (excellent):** Plus explicit distinction between *known unknowns* and *unknown unknowns*; quantified confidence intervals where applicable; awareness of model risk in own estimates.

## Scoring method

**Type:** marker search + structural assessment.

**Procedure:**

1. **Quantitative claim inventory.** Identify the report's quantitative claims (point estimates, projections, ratios). For each, check whether it is accompanied by an uncertainty marker from this lexicon:
   - Modality: `approximately`, `roughly`, `about`, `around`, `near`, `circa`, `~`
   - Probability: `likely`, `probably`, `may`, `could`, `is expected to`, `estimated to`, `projected to`
   - Range: `between X and Y`, `X to Y`, `X–Y`, `±`, `sensitivity range`
   - Epistemic status: `assume`, `assumption`, `under the scenario`, `if X holds`, `subject to`
2. **Limitations section check.** Is there an explicit section identifying limitations, caveats, or data quality issues?
3. **Calibration assessment.** Are hard, well-sourced facts (e.g., official statistics, court rulings) presented WITHOUT excessive hedging? Over-hedging known facts is also a calibration failure.

Score guidance:
- All quantitative claims appropriately hedged + limitations section + hard facts presented confidently → 8–10
- Most claims hedged + partial limitations awareness → 5–7
- Mixed: some hedged, some presented as certain → 3–5
- All claims presented as certain OR all claims over-hedged → 0–3

## Asymmetry per response type

- **AKM:** prompt mandates explicit data verification section with sources — expect uncertainty around projections, certainty around cited facts.
- **B1:** less structural pressure for limitations section; assess what's present in body text.
- **R1:** reports include a `Research Process Metadata` block with `confidence_level_pct` and `key_data_limitations`. **IGNORE this block.** Evaluate only the body of the report. Self-reported confidence does not contribute to scoring (flat matrix principle).

## R1 self-reported metadata handling

R1 reports may include `confidence_reported_pct` (e.g., 86, 95) and `key_data_limitations_reported`. **These do NOT count.** The body of the report must itself express calibration. If R1's body is confident but the metadata footer adds caveats, score the body only.
