# Criterion E5: Reproducibility Signals

## Definition

Does the report contain information that would allow another analyst to reproduce the reasoning? Assumptions, parameters, dates, data sources, calculation methodology.

This criterion is grounded in the reproducibility crisis literature for LLM evaluation (Belz, Thomson, Reiter, & Mille, 2023; Kosch & Feger, 2024) and the broader NLP reproducibility framework (Dodge et al., 2021). It operationalizes reproducibility at the *product* level — what's in the report — not at the *system* level (which is a separate concern).

E5 complements C5 (Process Auditability):
- **C5** asks: is the methodological path visible?
- **E5** asks: are the parameters available to replicate the calculation?

A report can score high on C5 (path shown) but low on E5 (path described without concrete parameters that would allow reproduction).

## Behavioral anchors (from BARS pilot v1)

- **Level 2 (poor):** No assumptions disclosed; black box.
- **Level 5 (average):** Some assumptions stated, most implicit.
- **Level 8 (good):** All 5 checklist items (below) covered. Observed in R1 (e.g., S1-R1-C line 43: explicit assumptions (`assumes no exemptions, no behavioural change, ordinary compliance`) + dates per source + method (turnover proxy from net sales) + parameters (96.5% sales-to-revenue ratio) + scope (2025 data)).
- **Level 10 (excellent):** Plus replication checklist + potential deviations. Observed in R1 (e.g., S2-R1-C lines 99–103: `key_data_limitations` with 4 explicit limitations + recommendation `should be tailored to a specific fund's liability profile`).

## Scoring method

**Type:** reproducibility checklist (adapted from Dodge et al., 2021, NLP Reproducibility Checklist).

**Procedure:** Score the report against the 5 items below, 2 points each (max 10).

### Item 1: Numerical assumptions disclosed (2 pts)
Are the numerical assumptions used in the analysis stated explicitly?
- Example: "GDP 2026 = PLN 3.5 trillion" — explicit
- Example: "we use median analyst forecast" — explicit (with reference)
- Counter-example: "given current macro conditions..." — implicit

### Item 2: Data source dates disclosed (2 pts)
Are the dates of the data sources used in the analysis stated?
- Example: "(Ministry of Finance, 2026)" — date present
- Example: "as of Q1 2026" — date present
- Counter-example: "according to recent data..." — date missing

### Item 3: Calculation method described (2 pts)
Is the method used for derived numbers (e.g., projections, ratios) described?
- Example: "we extrapolate using linear regression on 2018–2024 data" — described
- Example: "estimated using a turnover proxy from net sales of 10+-employee enterprises" — described
- Counter-example: "the projected value is X" — method missing

### Item 4: Model / framework parameters disclosed (2 pts)
Are the parameters of any analytical framework used (e.g., discount rate, time horizon, exchange rates) disclosed?
- Example: "discount rate 8%, 5-year horizon" — disclosed
- Example: "tax base assumed at 96.5% of total revenue per GUS 2025 ratio" — disclosed
- Counter-example: "NPV is positive" — parameters missing

### Item 5: Time scope of analysis disclosed (2 pts)
Is the time scope of the analysis (over what period) disclosed?
- Example: "we evaluate the policy over 2026–2031" — scope disclosed
- Example: "five-year horizon" — scope disclosed
- Counter-example: "in the long run..." — scope vague

### Bonus (optional, +1 if present): replication checklist
If the report includes an explicit replication note (e.g., "to replicate this analysis, one needs: ..."), add +1 to score, capped at 10.

## R1 self-reported metadata handling

R1 reports include `confidence_justification_reported` and `key_data_limitations_reported` in metadata. These count as REPRODUCIBILITY SIGNALS in the body of the report IF they appear in the body. If they only appear in a metadata block and not in the analytical narrative, do NOT count them.

The reason: a reader of the report should be able to see assumptions/limitations in the analysis itself, not have to navigate a separate metadata appendix.
